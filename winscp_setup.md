# MovingArts 프로젝트를 위한 WinSCP 설정 가이드

`Ubuntu-1` 인스턴스의 파일과 데이터베이스 설정을 직관적으로 관리하려면 다음 단계를 따르세요.

## 1. SSH 키 다운로드 (AWS Lightsail 콘솔)
1.  **AWS Lightsail 콘솔**에 접속합니다.
2.  우측 상단의 **Account** -> **Account**를 클릭합니다.
3.  **SSH Keys** 탭으로 이동합니다.
4.  인스턴스가 위치한 리전(예: 서울 - ap-northeast-2)의 **Default Key**를 다운로드합니다. 이 파일은 `.pem` 확장자입니다.

## 2. WinSCP 설정
1.  **WinSCP**를 실행합니다.
2.  **새 사이트(New Session)**를 클릭합니다.
3.  **파일 프로토콜**: `SFTP`를 선택합니다.
4.  **호스트 이름**: `3.36.149.255`를 입력합니다.
5.  **사용자 이름**: `ubuntu`를 입력합니다.
6.  **설정(Advanced...)** -> **SSH** -> **인증(Authentication)**으로 이동합니다.
7.  **개인 키 파일(Private key file)**에서 다운로드한 `.pem` 파일을 선택합니다. (WinSCP가 `.ppk` 형식으로 변환할지 묻는다면 **예(Yes)**를 클릭합니다.)
8.  **확인**을 누른 뒤, 세션 이름을 "MovingArts-Server" 등으로 **저장(Save)**합니다.
9.  **로그인**을 클릭합니다.

## 3. 주요 경로 탐색
- **프로젝트 파일**: `/home/ubuntu/movingarts_theory`
- **데이터베이스 설정**: `/etc/mysql/mysql.conf.d/mysqld.cnf`
- **SSD 마운트 포인트**: (이전 단계에서 확인한 경로를 사용하세요.)

## 5. 접속 오류 해결 (Connection Timed Out)
"Network error: Connection to '...' timed out" 오류가 발생한다면 다음을 확인하세요:

1.  **IP 주소 확인**: 호스트 이름이 `3.36.149.255` (또는 `.env`의 DB_HOST)로 접속하여 데이터를 직접 확인하세요.

## 5. 최신 코드 업로드 방법
서버에 수정한 코드를 반영하는 가장 쉬운 방법입니다.

### A. 드래그 앤 드롭 (추천)
1.  WinSCP 왼쪽 창(내 컴퓨터)에서 수정된 파일(예: `backend/src/...`)을 선택합니다.
2.  오른쪽 창(서버)의 적절한 위치(`/home/ubuntu/movingarts/...`)로 끌어다 놓습니다.
3.  "업로드" 확인창이 뜨면 **확인**을 누릅니다.

### B. 디렉토리 동기화 (고급)
변경사항이 많을 때 한 번에 반영하는 방법입니다.
1.  상단 툴바에서 **동기화(Synchronize)** 버튼(동그란 화살표 아이콘)을 누릅니다.
2.  **방향**: "원격(Remote)" (내 컴퓨터 -> 서버)을 선택합니다.
3.  **모드**: "새로 고침(Preview)"을 눌러 어떤 파일이 바뀔지 미리 확인한 뒤 실행합니다.

> [!IMPORTANT]
> **`node_modules` 폴더는 절대 업로드하지 마세요!**
> 이 폴더는 용량이 매우 커서 업로드 시 시간이 아주 오래 걸리고 오류가 날 수 있습니다. 서버 터미널에서 `npm install`을 실행하는 것이 정석입니다.

### C. 업로드 후 서버 적용
파일을 올린 후에는 서버 터미널에서 서비스를 재시작해야 반영됩니다.
```bash
pm2 restart all  # 또는 pm2 restart 0
```
2.  **포트 22 (SSH) 개방**:
    - AWS Lightsail 콘솔 -> 네트워킹 탭으로 이동합니다.
    - **IPv4 방화벽**에 **SSH (포트 22)** 규칙이 있는지 확인합니다.
    - 없다면 **규칙 추가**를 눌러 **Application: SSH**, **Protocol: TCP**, **Port: 22**를 추가하세요.
3.  **사용자 이름**: `ubuntu`가 맞는지 확인하세요 (root가 아님).
4.  **로컬 방화벽**: 회사나 공용 와이파이 환경에서는 22번 포트 접속이 차단될 수 있습니다. 테더링 등을 이용해 테스트해 보세요.
